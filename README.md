
# [프로젝트명: Jinary] **가칭: 고성능 웹 바이너리 통신 자동화 프레임워크**
## 1. 프로젝트 개요 (Project Overview)

* **팀 구성**
  * 👑 **정상화** (백엔드 개발자, 팀장)
  * 🧑‍💻 **권민석** (프론트엔드 개발자, 팀원)

* **주제 (Theme)**
  > 대 AI 시대, 데이터 하나의 용량이 커지고 많은 데이터가 오가고 있는 상황 속에서 **기존 JSON 기반의 비효율적인 통신 패러다임을 혁신**합니다. 대규모 데이터를 2진수(Binary) 형태로 초고속 전송하고, 이를 자동 처리하는 **개발자 친화적 통신 라이브러리**를 개발합니다.

<br/>

## 2. 배경 및 필요성 (Background)

* 📉 **JSON 통신의 한계**
  현재 웹 표준인 JSON은 텍스트 기반으로 이루어져 있어, 데이터 크기가 커질수록 네트워크 대역폭 낭비와 CPU 파싱 오버헤드가 기하급수적으로 증가합니다.

* 🤖 **AI 시대의 요구사항**
  AI 모델의 벡터 데이터, 고해상도 멀티미디어, 실시간 대시보드 등 전송 데이터의 단위가 커지는 추세입니다. 이러한 환경에서 기존의 텍스트 기반 방식으로는 최상의 퍼포먼스(Low Latency, High Throughput)를 구현하기 어렵다고 판단했습니다.

* 🚧 **기존 기술의 문제점 (Pain Point)**
  `gRPC-web` 등 기존의 바이너리 통신 기술이 존재하나, 환경 설정이 매우 복잡하고 프록시 서버(Proxy Server) 설치가 필수적으로 요구되어 실제 웹 개발 환경에서의 도입 장벽이 매우 높습니다.

<br/>

## 3. 주요 개발 내용 및 로직 (Core Features)

본 프로젝트는 개발자가 복잡한 인프라 설정(프록시 서버 등)이나 프로토콜 규격 작성 없이, **어노테이션(Annotation) 기반**으로 바이너리 통신을 즉시 구현할 수 있는 환경을 제공합니다.

* **목표:** 개발자 경험(DX)을 극대화한 **'Zero-Configuration'** 바이너리 브릿지 구축
